Setup

Folders

When you unpack the bundle, you'll get 2 folders

  1. Documentation
  2. Template

The documentation is only for your knowledge. The template contains all the HTML and Styling files.

Using the Template

To start up a local server and open the template, you will require NPM, which is the standard node package manager. You can download it from here.

If you wish to use the pages in your project (which doesn't require NPM), then you can do so aswell by just using the .html files and css/style.css file.

Setting up a local server

  1. Once you have installed npm, go to the Template folder directory using your terminal / command line interface, and run the command 'npm install' to install all the dependencies used in this project, including SASS which is used for writing the styling code.

  2. After all dependencies have been installed, run 'npm run start' to start the local development server. Once this command is run, you will be able to open up the template on http://localhost:8080 in your browser.

  3. Select the page you want to visit from the list shown there. To make changes, edit the .html files or .sass files directly and you will see the changes reflected automatically!

How To Update Styling

All of the .html pages import the final compiled CSS code from css/style.css file. There are corresponding .scss files which contain the specific styling for that page. If you want to change the styling, you can modify the corresponding .scss file and on saving it will autocompile and update the css/style.css file, and you will be able to see the reflected changes on your local server.

SASS / CSS Structure

If you are not familiar with SASS or CSS, you can learn about the basics of CSS here and SASS here

SASS and CSS Folder Structure

There are 2 types of styling files in this project. SASS files (with the extension .scss) and CSS files (with the extension .css). To make changes in the styling, edit only the SASS files. On saving the SASS files, the local npm server will autocompile the .scss code into .css and you will see the changes reflecting in your page. If you wish to directly use the html page and .css file in your project, then you can use the style.css file in the css folder, which contains the final compiled css.

The folder structure is as shown below

  • css
    1. style.css - contains the final compiled CSS

  • sass
    1. pages
      1. community.scss - contains styling for community.html
      2. create-post.scss - contains styling for create-post.html
      3. login.scss - contains styling for login.html
      4. post-details.scss - contains styling for post-details.html
      5. posts.scss - contains styling for posts.html
    2. utils
      1. common.scss - contains utility functions, variables and mixins
    3. main.scss - imports all the .scss files and contains global styling rules

Detailed comments are present in each .scss file with the table of contents.

General SASS File Structure

All the SASS files follow a simple order.

  1. Table of Contents - structure for the particular .scss file
    2.                                     -------------------------------------------------------------------*/
                                    /*------------------------------------------------------------------
                                    [Table of contents]

                                    1. Import styling of all pages
                                    2. Global styling rules
                                    3. Styling for common elements
                                        -3.1 Page Navigation
                                        -3.2 Page Footer
                                    4. Media Queries

                                    -------------------------------------------------------------------*/
  2. Imports: All styling related imports used in this file
    3.                                     /* Import styling of all pages */
                                    @import './pages/login.scss';
                                    @import './pages/posts.scss';
                                    @import './pages/post-details.scss';
                                    @import './pages/community.scss';
                                    @import './pages/create-post.scss';
  3. Page body: Contains all the styling related to the page

    4. The styling rules follow the BEM convention where class names are of the form block-name__elem-name. Therefore, you will see that all the components in every page have the class name as page-name__component-name. For example, to style the body for the Login page which has the class 'login-page__body', see below snippet. 

                                        .login-page {

                                        &__body {
                                            color: theme-text-color($selected-theme);
                                        }

                                        // ..style other content in this page..
                                    }
All the .scss files are documented properly so you can navigate through the content styling. names.
